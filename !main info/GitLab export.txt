Пошаговая инструкция:
    Инициализация Git в проекте (если еще не сделано):
        Откройте ваш проект в CLion.
        Откройте встроенный терминал (View -> Tool Windows -> Terminal).
        Выполните команды:
            bash
            git init
            git add .
            git commit -m "Initial commit"

    Создание репозитория в GitLab:
        Зайдите на ваш GitLab и создайте новый (пустой) проект.
        Скопируйте URL этого репозитория (HTTPS или SSH).
        Добавление удаленного репозитория (GitLab) в CLion:
        В терминале CLion, выполните:
            bash
            git remote add origin <URL вашего репозитория из GitLab>
                    (Например, git remote add origin gitlab.com)

    Отправка файлов в GitLab:
        Выполните команду для отправки локальных файлов в удаленный репозиторий:
            bash
            git push -u origin main
            (Или git push -u origin master, если ваша основная ветка называется master).

После этого ваши файлы появятся в репозитории на GitLab.
Для последующих изменений просто делайте git add, git commit, а затем git push.

Основное решение: Сначала синхронизируйте изменения (Pull)
    В большинстве случаев проблема решается путем получения последних изменений с GitLab и
    объединения их с вашими локальными изменениями.

Сделайте Pull (Получить изменения):
    В меню CLion выберите Git > Pull... (или нажмите Ctrl+T).
    Убедитесь, что выбран правильный удаленный репозиторий (origin) и нужная ветка (например, main или master).

    Нажмите кнопку Pull.

Разрешите конфликты слияния (если они возникнут):
    Если Git не сможет автоматически объединить изменения, появится окно Conflicts (Конфликты).
    Вам нужно будет вручную разрешить конфликты в соответствующих файлах
    (CLion предоставит удобный интерфейс для этого).

После разрешения конфликтов закоммитьте изменения слияния.
    Повторите Push (Отправить изменения):
    После успешного pull и разрешения конфликтов снова попробуйте отправить свои изменения: Git > Push (или Ctrl+Shift+K).

Другие возможные причины и решения
Если стандартный pull не помог, возможно, причина в следующем:
Защищенная ветка (Protected Branch): В Skillbox или администратором проекта на ветку
main (или master) могут быть установлены правила защиты, запрещающие прямую отправку (push) кода.

Решение: Создайте отдельную ветку для своей работы (например, git checkout -b my-feature-branch), отправьте изменения в нее (Push), а затем создайте Merge Request (Запрос на слияние) в веб-интерфейсе GitLab для объединения вашей ветки с основной.
У вас нет прав на запись: Убедитесь, что вы вошли в правильный аккаунт GitLab в CLion и у вас есть достаточные права доступа (роль Developer или выше) для отправки изменений в этот конкретный репозиторий.
Первый push в пустой репозиторий: Если вы пытаетесь сделать самый первый push в абсолютно пустой удаленный репозиторий (в котором даже нет файла README), иногда могут возникать проблемы.
Решение: Убедитесь, что вы сделали локальный коммит перед первым push'ем. Использование функции Git > GitLab > Share Project on GitLab обычно автоматически решает эту проблему.
